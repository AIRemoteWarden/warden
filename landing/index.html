FAQ

What is a temporary SSH access alternative?

It is a way to let someone help in a shell session without giving them direct SSH credentials, a permanent account, or a shared private key.

Is Warden the same as temporary SSH access?

No. Warden is an alternative to temporary SSH access. The goal is the same, which is short-term terminal collaboration, but the workflow keeps the host in control and avoids SSH handoff.

Why not just use SSH?

Because the real problem is usually not access. It is the trust cost of giving direct shell access to someone outside your team.

Why not just use Zoom or Slack screen sharing?

Because screen share is a visual workaround. It is slower when both sides need to work inside the shell.

Can the host stop the session immediately?

Yes. The host can interrupt and close the session locally.

How is Warden secured?

Warden is built around local host control, temporary sessions, approval gates, and trusted model integrations. End-to-end encryption is on the roadmap.

Can I self-host Warden?

Yes. Warden is designed to support self-hosted deployments for teams that want to run it in their own environment.
